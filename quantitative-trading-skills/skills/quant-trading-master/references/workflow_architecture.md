## 核心组件

### 1. QuantTradingContext（上下文对象）

负责存储任务的状态信息，包括：
- task_id: 任务唯一标识
- session_id: 会话唯一标识
- stock_pool: 股票池
- time_range: 时间范围
- artifacts: 已生成产物
- config: 配置信息
- current_stage: 当前阶段
- stage_history: 阶段历史
- results: 各阶段结果

### 2. StageStateMachine（阶段状态机）

定义了完整的投研流程和阶段转换逻辑：

```
数据获取 → 因子构建 → 模型训练 → 回测验证 → 组合优化 → 模拟／实盘 → 绩效报告
```

支持分支逻辑：
- 回测失败 → 返回因子调优
- 模型过拟合 → 样本外再验证

### 3. SubSkillDispatcher（子Skill调度器）

负责调用各子Skill完成具体工作。

### 4. MilestoneChecker（里程碑检查器）

在每个阶段完成后检查：
- 产物完整性
- 结果合理性
- 失败时给出明确错误

## 数据流转

1. 用户创建Context对象
2. 主Skill从初始阶段开始执行
3. 状态机调度对应子Skill
4. 子Skill执行并返回结果
5. 检查点验证
6. 进入下一阶段或分支
7. 重复直至完成

## 检查点机制

每个阶段完成后保存Context状态到磁盘，支持：
- 任务断点续传
- 结果回滚
- 任务恢复
